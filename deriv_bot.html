    ⚠️ Important — Read Before Trading Real Money

    Real-Money Failure Modes

    • Browser tabs can crash, be suspended, or lose network silently — bot stops, open contracts continue to expiry.
    • WebSocket disconnections mid-trade leave open contracts unmonitored.
    • Phone screen-lock or power-save can kill the page on mobile.
    • Slippage and proposal expiry can cause buy failures even with valid signals.
    • Past signal performance on buffered data does not predict future market conditions.

    Before Going Live

    • Run exclusively in Shadow Mode for at least 50+ trades.
    • Verify your token has trade scope but start with demo tokens.
    • Keep your risk limits tight — daily loss limit should be ≤5% of account.
    • Never leave a live bot unattended for more than a few minutes.
    • Use a dedicated Deriv demo account first.

    Early Sell / Contract Limitations

    • Not all Deriv contract types support early sell. Tick-based contracts often do NOT allow early close.
    • Even if is_valid_to_sell is true, the actual sell price will differ from displayed bid due to spread and latency.
    • The bot checks sell availability before attempting — if unavailable it waits for natural expiry.

    Strategy Adaptation Limitations

    • Adaptive weights require a minimum of 20 trades per indicator before adjusting.
    • Weight drift is capped at ±30% of base weight per session to prevent overfitting.
    • In-session adaptation does not carry over between page reloads.
    • High volatility regimes (ADX<15) trigger no-trade mode automatically.

    Browser Reliability

    • Use a desktop browser (Chrome/Firefox) for best reliability.
    • Disable power-save and screen-off if running on mobile.
    • Do not run multiple bots in different tabs with the same token.
    • This app uses no localStorage — all state is lost on page reload.
