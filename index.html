Simulation notes

Transparent limitations
This panel uses buffered market data and the current strategy settings to approximate trade outcomes. It cannot reproduce broker-side fills, pricing nuances, or every contract detail. Use it as a risk-check, not as proof of live expectancy.

Bot help and limitations

Real-money failure modes

Strategy failure can come from spread, execution timing, bad regime classification, sudden volatility shifts, stale data, API interruptions, and overfitting. The bot therefore blocks trades unless the readiness checklist passes.

Before going live

Test in demo or shadow mode first. Confirm the token has the required permissions, the symbol is correct, the duration is exact, and the risk caps are acceptable. Keep live trading disarmed until you are deliberately ready.

Early-sell / contract limitations

Not every contract can be closed early. This app checks the live open-contract state before attempting a sell. If the broker reports that a contract is not valid to sell, the sell button is disabled and the reason is shown instead of pretending otherwise.

Browser reliability limitations

Local browser execution depends on tab uptime, WebSocket stability, and device/network health. If the tab suspends or the connection drops, the app pauses new entries and logs the disruption. A browser bot is not a server process.

Strategy adaptation limitations

Adaptive weights are intentionally capped and decay slowly. The bot uses recent outcomes to nudge weights, but it does not assume markets are stationary and it does not chase recent wins aggressively.
